What You Get vs. What's Missing

Improved with Pure Go

  • Proper settings window instead of right-click menu
  • Real text inputs (no VBScript InputBox popups!)
  • Status bar showing URL/uptime
  • Dropdowns for provider selection
  • Better buttons for start/stop/logs
  • Stays at ~25 MB RAM

Cannot Achieve in Pure Go

  • Acrylic blur — technically possible on the window frame via DWM, but not inside
  • Fluent Design controls — no NavigationView, no Card UI, no modern typography
  • Dark mode — Go toolkits don't support it; you'd need custom draw everything
  • Animations / transitions — no motion design infrastructure
  • Reveal highlight, connected animations, parallax — none of the Fluent signature features

The Core Tradeoff

Pure Go improves usability (no more VBScript popups, actual window with fields). But it cannot deliver the WinUI 3 aesthetic — the ceiling is "clean Windows 7-era desktop app." To get acrylic, dark mode, and Fluent controls, you need WebView2.
