The WebView2 Reality

Cold Start: 40-80 MB

First launch with no cached Edge process. WebView2 spins up a full Chromium renderer. Comparable to opening a tab in Edge.

Warm State: 15-30 MB extra

After first launch, Edge keeps a shared GPU process. Subsequent launches share it — incremental cost is lower.

UI Complexity Matters

A simple settings page uses < 30 MB of the WebView2 budget. Heavy SPAs (React + charts + animations) push to 100+ MB.

Bottom Line

Current proxy uses ~15 MB. Any WebView2-based approach (Wails, Tauri) adds 60-160 MB — 10x increase. Pure Go stays light. The question is whether a 100MB settings window is worth it for a proxy that lives in the tray most of the time.
